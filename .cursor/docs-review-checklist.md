# ドキュメントレビューチェックリスト

ブログ記事や技術ドキュメントを作成・更新する際の品質チェックリストです。

## 実装コードとの整合性チェック

- [ ] シーケンス図やフロー図が実装コードと一致しているか
- [ ] 説明されている順序が実装コードの実行順序と一致しているか
- [ ] コード例が実際のコードベースから正しく引用されているか
- [ ] 実装が理論的な原則に反している場合は、その旨を明記しているか

## 理論と実装の区別

- [ ] 「何をやるのか」セクションでは理論的な説明をしているか
- [ ] 「技術的な詳細」セクションでは実装コードを引用しているか
- [ ] 理論と実装が異なる場合は、その違いを明確に説明しているか

## 技術的な正確性

- [ ] 専門用語の説明が正確か
- [ ] 歴史的な背景が正確か
- [ ] コード例が実際に動作するか（可能な範囲で）

## 読みやすさ

- [ ] 段階的に説明されているか
- [ ] 比喩が適切か
- [ ] 図解が分かりやすいか

## チェック方法

1. **実装コードの確認**: 説明されている機能の実装コードを必ず確認する
2. **シーケンス図の検証**: シーケンス図の順序が実装コードの実行順序と一致しているか確認する
3. **原則との照合**: 実装が理論的な原則に反している場合は、その旨を明記する
