description: 現在のブログ記事やドキュメントの内容を基に、技術的なツイートを生成する
model: haiku
---

## 引数

コマンド実行時に以下の引数を指定できます：

- **ファイルパス（オプション）**: 特定のブログ記事やドキュメントのパスを指定。指定がない場合は、現在開いているファイルまたは最近閲覧したブログ記事を使用
- **トピック（オプション）**: ツイートの焦点を絞るトピック（例: "Iteratorモデル"、"Volcanoモデル"、"実行エンジン"など）

例: `tweet database/docs/blogs/phase1-executor.md Iteratorモデル`

## 振る舞い

- 引数でファイルパスが指定されている場合はそのファイルを、指定がない場合は現在開いているブログ記事やドキュメント（特に`database/docs/blogs/`配下のファイル）の内容を確認する
- 引数でトピックが指定されている場合は、そのトピックに焦点を当てたツイートを生成する
- 記事の要点を100文字以内で簡潔にまとめたツイートを生成する
- 技術的な内容を分かりやすく、興味深い形で表現する
- 適切なハッシュタグを1つ付ける
- プロジェクトの進捗や学びを共有する形式にする
- ツイートは日本語で生成する
- 生成したツイートは、そのまま投稿できる形式で出力する

## 出力形式

生成したツイートは、以下の形式で出力してください：

```
【生成されたツイート】

[ツイート本文]

#自作DB #ハッシュタグ
```

## 注意事項

- 100文字以内に収める（ハッシュタグを含む）
- 技術的な正確性を保つ
- 過度に宣伝的にならない
- 読者の興味を引く内容にする
