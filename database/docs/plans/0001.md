# RDB機能要件とディレクトリ構成設計

## 1. RDBに必要な機能要件（段階的実装）

### フェーズ1: 最小限の機能（MVP）

- **SQLパーサ**: CREATE TABLE, INSERT, SELECT
- **ストレージ層**: テーブルデータのファイル永続化
- **テーブル管理**: テーブルの作成、データの挿入・検索
- **データ型**: INT, TEXT（基本型）

### フェーズ2: 永続化と整合性

- **WAL（Write-Ahead Logging）**: トランザクションログによる障害復旧
- **チェックポイント**: 定期的なデータ整合性確保
- **バッファマネージャ**: ページキャッシュによるI/O最適化

### フェーズ3: クエリ処理の高度化

- **インデックス**: B+木による高速検索
- **WHERE句の拡張**: 複数条件、範囲検索
- **JOIN**: テーブル結合（Nested Loop）

### フェーズ4: トランザクション管理

- **MVCC（Multi-Version Concurrency Control）**: 並行実行制御
- **分離レベル**: READ COMMITTED, REPEATABLE READ
- **ロック機構**: 行ロック、テーブルロック

### フェーズ5: 最適化と運用

- **クエリオプティマイザ**: 実行計画の最適化
- **統計情報**: クエリ最適化のためのメタデータ
- **VACUUM**: ガベージコレクション

## 3. 各層の責務と依存関係

```
┌─────────────────┐
│   CLI/REPL      │  ← ユーザーインターフェース
└────────┬────────┘
         │
┌────────▼────────┐
│   Database      │  ← エントリーポイント、セッション管理
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
┌───▼───┐ ┌──▼──────┐
│Parser │ │Executor │  ← SQL解析と実行
└───┬───┘ └──┬──────┘
    │        │
    │   ┌────┴────┐
    │   │         │
┌───▼───▼──┐ ┌──▼────────┐
│ Catalog  │ │ Optimizer │  ← メタデータと最適化
└───┬──────┘ └──┬────────┘
    │           │
┌───▼───────────▼──┐
│   Storage       │  ← データ永続化
│   - Heap        │
│   - Buffer      │
│   - WAL         │
└─────────────────┘
```

## 4. 実装の優先順位

1. **ストレージ層** (`storage/`): データの永続化基盤
2. **パーサ層** (`parser/`): SQL文の解析（既存コードを拡張）
3. **実行エンジン** (`executor/`): クエリの実行
4. **カタログ層** (`catalog/`): メタデータ管理
5. **WAL層** (`wal/`): トランザクションログ
6. **インデックス** (`index/`): 高速検索
7. **トランザクション** (`transaction/`): 並行制御
8. **オプティマイザ** (`optimizer/`): クエリ最適化
